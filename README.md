# HuggingFace NLP 학습 프로젝트

한국어 자연어 처리(NLP) 기반 텍스트 분류 모델을 학습하고 실습하는 프로젝트입니다. HuggingFace의 `sentence-transformers` 사전학습 임베딩 모델과 scikit-learn 분류기를 조합하여, 다양한 도메인의 한국어 텍스트를 분류하는 파이프라인을 구축합니다.

## 공통 기술 스택

3개 프로젝트 모두 동일한 파이프라인 구조를 사용합니다.

| 구분            | 사용 기술                                                               |
| --------------- | ----------------------------------------------------------------------- |
| 언어            | Python 3                                                                |
| 임베딩 모델     | `sentence-transformers/paraphrase-multilingual-mpnet-base-v2` (768차원) |
| 분류기          | scikit-learn `LogisticRegression` vs `SVC` 비교 → 최선 자동 선택        |
| 주요 라이브러리 | sentence-transformers, scikit-learn, torch, pandas, numpy, matplotlib   |

### 공통 파이프라인

```
원본 텍스트 → 전처리 → 임베딩 벡터(768차원) → 분류기 학습 → 평가(Accuracy, 분류 리포트, 혼동 행렬)
```

1. 데이터 로드 및 전처리 (정제, 라벨 매핑, 균등 샘플링)
2. `paraphrase-multilingual-mpnet-base-v2`로 문장 임베딩 생성
3. `LogisticRegression`과 `SVC(RBF)` 두 분류기를 동시에 학습
4. 정확도 비교 후 최선 분류기를 자동 선택
5. 분류 리포트와 혼동 행렬 시각화로 성능 분석
6. 새 텍스트 입력 → 분류 결과 + 확률 출력

## 프로젝트 상세

### 1. 고객문의 분류기

K쇼핑 콜센터 질의응답 데이터를 활용하여, 고객 문의 텍스트를 **6개 유형**(AS, 결제, 교환, 반품, 배송, 주문)으로 자동 분류합니다.

| 항목        | 내용                                                |
| ----------- | --------------------------------------------------- |
| 분류 유형   | 다중 클래스 (6개 카테고리)                          |
| 데이터 출처 | AI Hub 민원(콜센터) 질의응답 데이터 -- K쇼핑 도메인 |
| 데이터 건수 | 카테고리별 1,000건 학습 / 200건 테스트              |
| 최종 분류기 | 로지스틱 회귀                                       |
| Accuracy    | **73.9%**                                           |

- v1(60.0%) → v2(63.6%) → v3(73.9%)까지 3단계에 걸쳐 모델을 개선
- 임베딩 모델 업그레이드(384→768차원), 카테고리 재정의(7→6개), 데이터 보강 등 개선 과정을 상세히 기록

자세한 내용: [고객문의\_분류기/README.md](고객문의_분류기/README.md)

---

### 2. 이커머스 리뷰 감성 분석기

Steam 게임 리뷰와 네이버 쇼핑 리뷰를 합쳐, 리뷰 텍스트를 **긍정/부정**으로 자동 분류합니다.

| 항목        | 내용                                                       |
| ----------- | ---------------------------------------------------------- |
| 분류 유형   | 이진 분류 (긍정/부정)                                      |
| 데이터 출처 | Steam 게임 리뷰 (100,000건) + 네이버 쇼핑 리뷰 (200,000건) |
| 데이터 건수 | 통합 300,000건 (긍정:부정 = 약 50:50)                      |
| 최종 분류기 | 로지스틱 회귀                                              |
| Accuracy    | **81.8%**                                                  |

- 두 개의 서로 다른 도메인(게임, 쇼핑) 리뷰를 합쳐 크로스 도메인 일반화 능력을 확인
- 명확한 감성 표현에 높은 확신도를 보이나, 반어법과 중립적 텍스트에서 한계 존재

자세한 내용: [이커머스*리뷰*데이터\_분석/README.md](이커머스_리뷰_데이터_분석/README.md)

### 3. 부서별 요청사항 라우팅 분류기

한국어 IT 헬프데스크 문의를 입력하면 **담당 부서(5개 카테고리)**를 자동으로 분류합니다.

| 항목        | 내용                                             |
| ----------- | ------------------------------------------------ |
| 분류 유형   | 다중 클래스 (5개 카테고리)                       |
| 데이터 출처 | `make_data.py`로 조합형 템플릿 기반 생성         |
| 데이터 건수 | 20,000건 (300+ 고유 패턴 조합, 100% 고유 텍스트) |
| 최종 분류기 | SVM (RBF)                                        |
| Accuracy    | **99.9%**                                        |

- 5개 카테고리: Hardware, Software, Inquiry/Help, Network, Database
- 99.9%의 정확도는 템플릿 기반 생성 데이터의 규칙적 패턴에 기인하며, 실무 환경에서의 성능과는 차이가 있음

자세한 내용: [부서별*요청사항*라우팅/README.md](부서별_요청사항_라우팅/README.md)

## 프로젝트 비교

| 항목        | 고객문의 분류기               | 이커머스 감성 분석기              | IT 헬프데스크 분류기 |
| ----------- | ----------------------------- | --------------------------------- | -------------------- |
| 분류 유형   | 다중 클래스 (6개)             | 이진 분류 (2개)                   | 다중 클래스 (5개)    |
| 데이터 출처 | K쇼핑 콜센터 (실제 데이터)    | Steam + 네이버 쇼핑 (실제 데이터) | 템플릿 기반 생성     |
| 데이터 규모 | 6,000건 학습 / 1,200건 테스트 | 300,000건 통합                    | 20,000건 생성        |
| 최종 분류기 | 로지스틱 회귀                 | 로지스틱 회귀                     | SVM (RBF)            |
| Accuracy    | 73.9%                         | 81.8%                             | 99.9%                |
| 성능 신뢰도 | 높음                          | 높음                              | 낮음 (생성 데이터)   |

### 성능 차이의 원인

- **고객문의 분류기 (73.9%)**: 실제 콜센터 대화 데이터로, "주문"과 "결제", "교환"과 "반품"처럼 의미적으로 유사한 카테고리 간 혼동이 발생. 현실적인 수준의 정확도
- **이커머스 감성 분석기 (81.8%)**: 이진 분류이므로 상대적으로 단순한 문제. 30만 건의 충분한 학습 데이터와 명확한 감성 표현이 성능을 끌어올림
- **IT 헬프데스크 분류기 (99.9%)**: 템플릿 기반 생성 데이터이므로 각 카테고리의 텍스트 패턴이 규칙적. 모델이 패턴을 학습한 것이지 실제 의미를 이해한 것이 아님
