# 이커머스 리뷰 감성 분석기

Steam 게임 리뷰와 네이버 쇼핑 리뷰를 합쳐, 리뷰 텍스트를 **긍정/부정**으로 자동 분류하는 AI 감성 분석기입니다.

## 프로젝트 구조

```
이커머스_리뷰_데이터_분석/
├── 이커머스_리뷰_데이터_분석.ipynb   # 메인 노트북 (전처리 → 학습 → 예측)
├── README.md
└── Data/
    ├── naver_shopping.txt           # 네이버 쇼핑 리뷰 원본 (200,000건)
    ├── steam.txt                    # Steam 게임 리뷰 원본 (100,000건)
    └── reviews_merged.csv           # 두 데이터를 합친 통합 파일 (300,000건)
```

## 사용 데이터

| 데이터           | 건수          | 라벨 기준                     | 형식                 |
| ---------------- | ------------- | ----------------------------- | -------------------- |
| Steam 게임 리뷰  | 100,000건     | 원본 라벨 (0=부정, 1=긍정)    | `라벨\t텍스트`       |
| 네이버 쇼핑 리뷰 | 200,000건     | 별점 1-2→부정(0), 4-5→긍정(1) | `별점\t텍스트`       |
| **합계**         | **300,000건** | 이진 분류 (0/1)               | `reviews_merged.csv` |

- 별점 3(중립)은 감성이 모호하므로 네이버 쇼핑 데이터에서 제외
- 긍정/부정 비율: 약 50:50 (149,959:150,041)으로 균등

## 모델 성능

### 분류기 비교 결과

| 분류기            | Accuracy           |
| ----------------- | ------------------ |
| **로지스틱 회귀** | **81.8%** (선택됨) |
| SVM (RBF)         | 80.5%              |

### 분류 리포트 (로지스틱 회귀)

| 클래스   | Precision | Recall   | F1-Score | Support   |
| -------- | --------- | -------- | -------- | --------- |
| 부정 (0) | 0.80      | 0.85     | 0.82     | 500       |
| 긍정 (1) | 0.84      | 0.79     | 0.81     | 500       |
| **전체** | **0.82**  | **0.82** | **0.82** | **1,000** |

### 혼동 행렬

|                | 예측: 부정 | 예측: 긍정 |
| -------------- | ---------- | ---------- |
| **실제: 부정** | 423 (TN)   | 77 (FP)    |
| **실제: 긍정** | 105 (FN)   | 395 (TP)   |

- **부정 정확도: 84.6%** (423/500) -- 부정 리뷰를 잡아내는 능력이 더 높음
- **긍정 정확도: 79.0%** (395/500) -- 긍정 리뷰 105건을 부정으로 잘못 분류
- False Negative(105)가 False Positive(77)보다 많아, 긍정을 부정으로 오분류하는 경향이 존재

## 모델 평가

### 강점

1. **명확한 감성에 높은 확신도** -- 명백한 긍정/부정 리뷰에 대해 95%+ 확률로 정확하게 분류
2. **크로스 도메인 일반화** -- 쇼핑 리뷰와 게임 리뷰 두 도메인 모두에서 작동
3. **문맥 기반 판단 가능** -- "처음에는 재미있었는데... 돈이 아깝네요"처럼 긍정 단어가 포함된 부정 리뷰를 올바르게 분류
4. **균형 잡힌 성능** -- F1-score가 부정(0.82), 긍정(0.81)으로 클래스 간 편향이 거의 없음

### 한계점

1. **반어법 인식 불가** -- "역시 최고입니다"를 83% 확률로 긍정으로 분류. 임베딩 모델이 표면적 의미에 의존하여 반어법을 감지하지 못함
2. **81.8% 정확도의 의미** -- 이진 분류에서 81.8%는 보통 수준. 100건 중 약 18건을 잘못 분류하므로 실무 적용 시 신뢰도 보완 필요
3. **게임 리뷰에 대한 상대적 낮은 확신도** -- 쇼핑 긍정 리뷰(99.5%)에 비해 게임 긍정 리뷰(83.2%)의 확신도가 낮음. 학습 데이터에서 네이버 쇼핑이 20만건 vs Steam 10만건으로 쇼핑 도메인에 편향
4. **학습 데이터 비율 불균형** -- 전체 30만건에서 5,000건만 샘플링(1.7%)하여 학습. 더 많은 데이터를 활용하면 성능 향상 가능

## 데이터 전처리 파이프라인

이커머스 리뷰는 사용자가 직접 작성한 텍스트이므로 다음과 같은 전처리를 적용합니다:

1. **HTML 태그 제거** -- 웹 크롤링 잔여물 (`<br>`, `<b>` 등) 정규식으로 제거
2. **반복 문자 축소** -- "ㅋㅋㅋㅋㅋ" → "ㅋㅋ" (3회 이상 반복을 2회로)
3. **공백 정리** -- 다중 공백 → 단일 공백, 앞뒤 공백 제거
4. **최소 길이 필터링** -- 5자 미만 텍스트 제거 (감성 판단 단서 부족)
5. **중복 제거** -- 동일 텍스트 제거 (과적합 방지)
6. **균등 샘플링** -- 라벨당 2,500건(학습용), 500건(평가용)

**전처리 결과:**

- 원본: 300,000건 → 전처리 후: 297,641건 (5자 미만 2,175건 + 중복 184건 제거)
- 학습 데이터: 5,000건 (부정 2,500 + 긍정 2,500)
- 평가 데이터: 1,000건 (부정 500 + 긍정 500)

## 모델 구조

```
리뷰 텍스트
    ↓ clean_text() 전처리
    ↓ SentenceTransformer 임베딩 (768차원 벡터, MPNet)
    ↓ LogisticRegression vs SVC 비교 → 최선 선택
    ↓
예측 결과: 긍정/부정 + 확률
```

```bash
pip install sentence-transformers scikit-learn torch matplotlib
```

### 2. 데이터 확인

아래 경로에 데이터 파일이 있는지 확인합니다:

```
이커머스_리뷰_데이터_분석/Data/
├── naver_shopping.txt     # 네이버 쇼핑 리뷰 (별점\t텍스트)
```

`reviews_merged.csv`가 없는 경우, `naver_shopping.txt`와 `steam.txt`를 아래 규칙으로 합칩니다:

- Steam: 라벨(0/1) 그대로 사용
- Naver Shopping: 별점 1-2 → 0(부정), 별점 4-5 → 1(긍정), 별점 3은 제외

### 3. 노트북 실행

VS Code 또는 Jupyter 환경에서 `이커머스_리뷰_데이터_분석.ipynb`를 열고, 위에서부터 순서대로 모든 셀을 실행합니다.

```
VS Code: Ctrl+Shift+P → "Run All" 또는 셀별로 Shift+Enter

```

MPNet 모델은 처음 실행 시 다운로드가 필요합니다 (약 1GB).
임베딩 처리 시간: train(5,000건) 약 30초, test(1,000건) 약 5초 (CPU 기준).

## 예측 사용 예시

노트북의 모든 셀 실행 후, `predict_sentiment()` 함수로 감성을 예측할 수 있습니다:

```python
predict_sentiment("배송도 빠르고 품질도 좋아요. 가격 대비 만족합니다.")
# → 긍정 (0.995)

predict_sentiment("사진이랑 너무 다르고 품질이 별로예요. 돈 아까워요.")
# → 부정 (0.997)

predict_sentiment("그래픽이 미쳤고 스토리 몰입감이 장난 아닙니다.")
# → 긍정 (0.832)
```

입력 텍스트에도 학습 시와 동일한 전처리(`clean_text()`)가 자동 적용됩니다.
