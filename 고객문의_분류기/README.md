# 🛒 고객 문의 유형 분류기

K쇼핑 콜센터 질의응답 데이터를 활용하여, 고객 문의 텍스트를 **7개 유형**(AS, 결제, 교환, 반품, 배송, 업무처리, 주문)으로 자동 분류하는 AI 분류기입니다.

## 프로젝트 구조

```
고객문의_분류기/
├── 고객_문의_유형_분류기.ipynb   # 메인 노트북 (전처리 → 학습 → 예측)
└── README.md
```

## 사용 데이터

- **출처:** AI Hub 민원(콜센터) 질의응답 데이터셋 — K쇼핑 도메인
- **위치:** `../Data/1.Training/라벨링데이터_231222_add/`, `../Data/2.Validation/라벨링데이터_231222_add/`
- **형식:** ZIP → JSON (대화셋 기반 콜센터 통화 녹취 텍스트)
- **카테고리:** AS, 결제, 교환, 반품, 배송, 업무처리, 주문 (7개)

## 데이터 전처리 파이프라인

원본 데이터는 콜센터 통화 녹취 텍스트이므로 다음과 같은 전처리를 적용합니다:

1. **대화셋 단위 결합** — 개별 문장이 아닌 `대화셋일련번호` 기준으로 고객 발화를 하나로 합침
2. **ㅇㅇㅇ 익명화 토큰 제거** — 정규식 `ㅇ{2,}` 패턴 제거
3. **필러 표현 제거** — "네.", "아 그래요?", "여보세요?" 등 분류에 무관한 반응형 표현 9종 제거
4. **최소 길이 필터링** — 15자 미만 텍스트 제거 (분류 단서 부족)
5. **중복 제거** — 동일 텍스트 제거
6. **균등 샘플링** — 카테고리별 500건(학습), 100건(테스트)

## 모델 구조

```
고객 문의 텍스트
    ↓ clean_text() 전처리
    ↓ SentenceTransformer 임베딩 (384차원 벡터)
    ↓ LogisticRegression 분류
    ↓
예측 결과: 카테고리 + 확률 (Top-3)
```

## 예측 사용 예시

노트북의 모든 셀 실행 후, `predict_inquiry()` 함수로 문의 유형을 예측할 수 있습니다:

```python
predict_inquiry("주문한 물건이 아직 안 왔어요. 배송 조회 좀 해주세요.")
# → 배송 (0.xxx)

predict_inquiry("이 제품 환불하고 싶어요. 반품 접수해주세요.")
# → 반품 (0.xxx)
```

입력 텍스트에도 학습 시와 동일한 전처리(`clean_text()`)가 자동 적용됩니다.
